# 🧾 **ARMテンプレートとは**

## 🔰 概要

**ARMテンプレート（Azure Resource Manager Template）** とは

👉 **Azureリソースをコードで定義してデプロイする仕組み**

---

## 💡 イメージ

```
手動（Portal）
→ クリックで作成

ARMテンプレート
→ コードで一括作成
```

---

## 🎯 試験ポイント（超重要）

- **インフラの自動化（IaC: Infrastructure as Code）**
- **同じ環境を何度でも再現できる**

---

# 🧠 **なぜ使うのか**

## ❌ 手動（Portal）

- ミスが起きやすい
- 再現性がない

---

## ✅ ARMテンプレート

- 一貫性あり
- 自動化可能
- 再利用可能

---

# 🧱 **テンプレートの構造（超重要）**

ARMテンプレートは **JSON形式** です

基本構造👇

```
{
  "$schema":"...",
  "contentVersion":"1.0.0.0",
  "parameters": {},
  "variables": {},
  "resources": [],
  "outputs": {}
}
```

---

## 各要素の意味

| 要素 | 内容 |
| --- | --- |
| **parameters** | 外部から値を渡す |
| **variables** | 内部で使う変数 |
| **resources** | 作成するリソース |
| **outputs** | デプロイ後の結果 |

---

## 🎯 試験ポイント

👉 **resources が最重要**

---

# 🛠️ **具体例（ストレージ作成）**

```
{
  "$schema":"https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion":"1.0.0.0",
  "resources": [
    {
      "type":"Microsoft.Storage/storageAccounts",
      "apiVersion":"2022-09-01",
      "name":"mystorageaccount123",
      "location":"japaneast",
      "sku": {
        "name":"Standard_LRS"
      },
      "kind":"StorageV2",
      "properties": {}
    }
  ]
}
```

---

## 💡 重要項目解説

- `"type"` → リソース種類
- `"name"` → リソース名
- `"location"` → リージョン
- `"sku"` → プラン（性能・冗長性）

👉 試験でよく問われる

---

# 🚀 **デプロイ方法（CLI）**

```
az deployment group create \
--resource-group myRG \
--template-file template.json
```

---

## 💡 オプション解説

| オプション | 内容 |
| --- | --- |
| `--resource-group` | デプロイ先 |
| `--template-file` | テンプレートファイル |

---

# 🧠 **parameters の使い方（超重要）**

## ■ 例

```
"parameters": {
  "storageName": {
    "type":"string"
  }
}
```

👉 外部から値を渡す

---

## CLIで値を渡す

```
az deployment group create \
--resource-group myRG \
--template-file template.json \
--parametersstorageName=mystorage123
```

---

## 🎯 試験ポイント

- **parameters = 柔軟性を持たせるため**

---

# 🔗 **依存関係（dependsOn）**

## ■ 例

```
"dependsOn": [
"[resourceId('Microsoft.Network/networkInterfaces', 'myNIC')]"
]
```

👉 「先にNICを作ってからVMを作る」

---

## 🎯 試験ポイント

- **リソースの作成順を制御**

---

# 🔥 **Bicepとの違い（超重要）**

## ■ Bicepとは

👉 ARMテンプレートを簡単に書ける言語

---

## 比較

| 項目 | ARMテンプレート | Bicep |
| --- | --- | --- |
| 形式 | JSON | 独自言語 |
| 可読性 | 低い | 高い |
| 学習難易度 | 高い | 低い |
| 試験重要度 | 高い | 高い |

---

## 💡 試験ポイント

- **BicepはARMに変換される**
- 実体はARM

---

# ⚠️ **よく出る引っかけ**

| 問題 | 正解 |
| --- | --- |
| ARMテンプレートはGUI専用 | ❌ |
| JSON以外で書けない | ❌（Bicepあり） |
| 手動の方が安全 | ❌（自動化の方が安全） |

---

# 🎯 **まとめ**

- **ARMテンプレート = インフラをコード化**
- JSON形式で記述
- 主な構成：
    - **parameters**
    - **variables**
    - **resources**
- CLIでデプロイ可能
- **Bicepは簡易版**

---

# 📊 **比較まとめ（Notion対応）**
