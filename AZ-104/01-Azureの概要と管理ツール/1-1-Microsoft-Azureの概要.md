# ☁️ **Azureの概要**

## 🔰 **Azureとは何か**

**Azure** は Microsoft が提供する **クラウドコンピューティングサービス**です。

👉 簡単に言うと

**「サーバーやネットワークを自分で持たずに、インターネット経由で利用できる仕組み」**

---

## 💡 **クラウドの特徴（試験頻出）**

| 項目 | 内容 |
| --- | --- |
| **従量課金** | 使った分だけ支払う |
| **スケーラビリティ** | 必要に応じて拡張・縮小できる |
| **高可用性** | 障害に強い構成が可能 |

👉 試験では

**「オンプレミスとの違い」** がよく問われます

---

# 🧱 **Azureの基本構造（超重要）**

Azureは以下の階層構造になっています👇

```
テナント（Microsoft Entra ID）
 └ サブスクリプション
    └ リソースグループ
       └ リソース
```

---

## ① **テナント（Microsoft Entra ID）**

- 組織の **ID管理基盤**
- ユーザー・グループを管理

👉 試験ポイント

- **認証（ログイン）を担当**

---

## ② **サブスクリプション**

- Azureの **課金単位**

👉 試験ポイント

- **コスト管理の単位**
- 1つのテナントに複数作成可能

---

## ③ **リソースグループ**

- リソースをまとめる単位

👉 特徴

- 同じライフサイクルで管理（作成・削除）

---

## ④ **リソース**

- 実際に使うサービス

例：

- **仮想マシン（VM）**
- **ストレージアカウント**
- **仮想ネットワーク**

---

# 🧠 **サービスモデル（超頻出）**

## ■ **IaaS（Infrastructure as a Service）**

- 仮想マシンなど
- OSから上は自分で管理

例：**Virtual Machines**

---

## ■ **PaaS（Platform as a Service）**

- アプリ実行環境
- OS管理不要

例：**App Service**

---

## ■ **SaaS（Software as a Service）**

- 完成済みサービス
- すぐ利用可能

例：Microsoft 365

---

## ⚠️ **責任範囲の違い（試験でよく出る）**

| 項目 | IaaS | PaaS | SaaS |
| --- | --- | --- | --- |
| OS管理 | 自分 | Azure | Azure |
| ミドルウェア | 自分 | Azure | Azure |
| アプリ | 自分 | 自分 | 不要 |

👉 問題例

「OSのパッチ適用は誰がやるか？」 → **IaaSは自分**

---

# 🌍 **リージョンと可用性（重要）**

## ■ **リージョン**

- データセンターの場所

例：`japaneast`

---

## ■ **可用性ゾーン**

- 同一リージョン内の別データセンター

👉 試験ポイント

- **高可用性（HA）を実現**

---

# 🛠️ **Azureの操作方法**

## ■ **Azure CLI（試験でも重要）**

ログイン：

`az login`

サブスクリプション一覧：

`az account list`

サブスクリプション切替：

`az account set --subscription "<サブスクリプション名>"`

---

## 💡 コマンドのポイント

- `az` = Azure CLIのコマンド
- `account` = サブスクリプション操作
- `set` = 切り替え

👉 試験では

**「どのコマンドで何ができるか」** が問われます

---

# ⚠️ **試験の引っかけポイント**

| 間違い | 正解 |
| --- | --- |
| リソースグループ単位で課金 | **サブスクリプション単位** |
| リージョンとゾーンは同じ | **別物** |
| PaaSでもOS管理が必要 | **不要** |

---

# 🎯 **まとめ**

- **Azure** = クラウドサービス
- 階層構造**テナント → サブスクリプション → リソースグループ → リソース**
- サービスモデル**IaaS / PaaS / SaaS**
- 操作方法**Portal / CLI / PowerShell**

---

# 📊 **比較まとめ（Notion貼り付けOK）**

## ■ 階層構造

| 項目 | 役割 |
| --- | --- |
| **テナント** | ID管理（Entra ID） |
| **サブスクリプション** | 課金単位 |
| **リソースグループ** | リソース管理 |
| **リソース** | 実体（VMなど） |

---

## ■ サービスモデル比較

| モデル | 特徴 | 管理範囲 |
| --- | --- | --- |
| **IaaS** | 自由度が高い | OS以上を管理 |
| **PaaS** | 開発に集中 | アプリのみ |
| **SaaS** | 完成済み | 管理不要 |
