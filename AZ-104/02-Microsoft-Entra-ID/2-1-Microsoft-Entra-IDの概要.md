# 🔐 **Microsoft Entra ID の概要**

## 🔰 **Microsoft Entra IDとは**

**Microsoft Entra ID** は

👉 **AzureのID管理サービス（認証基盤）**

---

## 💡 一言でいうと

👉 **「誰がログインできるか」を管理する仕組み**

---

## ⚠️ 旧名称

- 旧：**Azure Active Directory（Azure AD）**
- 新：**Microsoft Entra ID**

👉 試験では両方出る可能性あり

---

# 🧠 **役割（超重要）**

## ① **認証（Authentication）**

👉 「本人確認」

例：

- ユーザー名 + パスワード
- 多要素認証（MFA）

---

## ② **認可（Authorization）との関係**

👉 **Entra ID単体では認可しない**

- 認証 → **Entra ID**
- 認可 → **RBAC**

---

## 🎯 試験ポイント（超重要）

👉 この違いは必ず出る

---

# 👤 **管理できるもの**

## ■ ユーザー

- 個人アカウント

---

## ■ グループ

- 複数ユーザーをまとめる

---

## ■ サービスプリンシパル

👉 アプリ用のID

---

## ■ マネージドID

👉 Azureリソース用のID

---

## 💡 試験ポイント

- **サービスプリンシパル vs マネージドID** の違い

---

# 🧱 **テナント（Tenant）**

## 🔰 概要

**Entra ID = テナント**

👉 組織単位のID管理領域

---

## 💡 特徴

- ユーザー・グループを格納
- サブスクリプションと紐づく

---

## 🎯 試験ポイント

- **1テナントに複数サブスクリプション**

---

# 🔑 **認証方法**

## ■ パスワード認証

基本のログイン

---

## ■ 多要素認証（MFA）

- セキュリティ強化

---

## ■ 条件付きアクセス

👉 条件でアクセス制御

例：

- 社内IPのみ許可
- デバイス制限

---

## 🎯 試験ポイント

- **MFAはセキュリティ強化の基本**
- **条件付きアクセスは頻出**

---

# 🔗 **RBACとの関係（最重要）**

| 項目 | 役割 |
| --- | --- |
| **Entra ID** | 認証（ログイン） |
| **RBAC** | 認可（権限付与） |

---

## 💡 イメージ

```
ログイン → Entra ID
操作権限 → RBAC
```

---

## ⚠️ 試験の引っかけ

❌ Entra IDが権限管理する

👉 → **RBACが管理**

---

# 🛠️ **CLI操作（基本）**

## ■ ユーザー作成

```
az ad user create \
--display-name"testuser" \
--user-principal-name testuser@yourdomain.com \
--password"Password123!"
```

---

## ■ グループ作成

```
az ad group create \
--display-name"testgroup" \
--mail-nickname"testgroup"
```

---

## 💡 コマンドポイント

- `az ad` → Entra ID操作
- `user` → ユーザー管理
- `group` → グループ管理

---

# 🔥 **よく出る比較（超重要）**

## ■ サービスプリンシパル vs マネージドID

| 項目 | サービスプリンシパル | マネージドID |
| --- | --- | --- |
| 用途 | アプリ認証 | Azureリソース |
| 管理 | 手動 | 自動 |
| シークレット | 必要 | 不要 |

---

# 🎯 **まとめ**

- **Microsoft Entra ID = ID管理サービス**
- 役割：
    - **認証（ログイン）**
- 管理対象：
    - ユーザー / グループ / アプリ
- RBACとの違い：
    - **Entra ID → 認証**
    - **RBAC → 認可**

---

# 📊 **比較まとめ（Notion対応）**

## ■ 認証と認可

| 項目 | 内容 |
| --- | --- |
| **認証（Authentication）** | 本人確認（Entra ID） |
| **認可（Authorization）** | 権限管理（RBAC） |

---

## ■ ID種類

| 種類 | 用途 |
| --- | --- |
| **ユーザー** | 人 |
| **グループ** | 複数ユーザー |
| **サービスプリンシパル** | アプリ |
| **マネージドID** | Azureリソース |
