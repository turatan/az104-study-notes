# 🔐 **ネットワーク セキュリティ グループ（NSG）**

ここはAZ-104ネットワーク分野の超重要ポイントです 👍

試験でもかなり頻出です。

特に👇が重要です

- **通信許可 / 拒否**
- **ポート制御**
- **優先順位**
- **サブネット / NIC適用**

---

# 🔰 **NSGとは**

## ■ 正式名称

**Network Security Group**

---

## ■ 概要

👉 **Azureの通信制御ファイアウォール**

---

## 💡 一言でいうと

👉 **「どの通信を通すか決めるルール」**

---

# 🧠 **イメージ（超重要）**

```
インターネット
      ↓
NSG
      ↓
VM
```

---

## 🎯 試験ポイント

- NSG = パケットフィルタ
- 通信の許可・拒否

---

# 🌐 **NSGで制御できるもの**

| 項目 | 内容 |
| --- | --- |
| **送信元IP** | 誰から |
| **送信先IP** | どこへ |
| **ポート** | 何番 |
| **プロトコル** | TCP / UDP |
| **方向** | 受信 / 送信 |

---

# 🎯 試験ポイント

👉 「通信条件」を細かく制御可能

---

# 🔥 **NSGルール（超重要）**

---

# 🔰 基本構造

```
条件
 ↓
許可（Allow） or 拒否（Deny）
```

---

# 🧠 **例**

| 条件 | 動作 |
| --- | --- |
| `22/TCP` | SSH許可 |
| `3389/TCP` | RDP許可 |
| `80/TCP` | HTTP許可 |

---

# 🎯 試験ポイント

👉 Linux接続 → `22`

👉 Windows接続 → `3389`

---

# ⚖️ **受信（Inbound）と送信（Outbound）**

| 種類 | 内容 |
| --- | --- |
| **Inbound** | 外から入る通信 |
| **Outbound** | 外へ出る通信 |

---

# 🎯 試験ポイント

👉 SSH/RDPはInbound

---

# 🏷️ **優先順位（超頻出）**

---

# 🔰 概要

NSGは番号順で評価

---

## 💡 重要

👉 数字が小さいほど優先

---

## 🧠 例

| 優先度 | 動作 |
| --- | --- |
| `100` | Allow SSH |
| `200` | Deny All |

---

## 🎯 試験ポイント

👉 最初に一致したルールが適用

---

# ⚠️ 超重要

後続ルールは見ない

---

# 🔥 **既定ルール（Default Rules）**

Azureには初期ルールがあります。

---

# 🧠 例

| ルール | 内容 |
| --- | --- |
| `AllowVNetInBound` | VNet内通信許可 |
| `DenyAllInbound` | その他拒否 |

---

## 🎯 試験ポイント

👉 最後にDenyが存在

---

# 📍 **NSG適用場所（超重要）**

---

# ① 📦 サブネット適用

```
Subnet
  ↓
NSG
```

👉 サブネット全体へ適用

---

# ② 🖥️ NIC適用

```
VM
 ↓
NIC
 ↓
NSG
```

👉 VM単位制御

---

# 🎯 試験ポイント

👉 NSGは両方適用可能

---

# ⚠️ **両方に適用時（重要）**

👉 両方のルールを通過必要

---

# 🛠️ **NSG作成（CLI）**

---

## ■ NSG作成

```
az network nsg create \
--resource-group myRG \
--name myNSG
```

---

# 🛠️ **ルール追加**

---

## ■ SSH許可

```
az network nsg rule create \
--resource-group myRG \
--nsg-name myNSG \
--name AllowSSH \
--priority100 \
--direction Inbound \
--access Allow \
--protocol Tcp \
--destination-port-ranges22
```

---

# 💡 オプション解説

| オプション | 内容 |
| --- | --- |
| `--priority` | 優先順位 |
| `--direction` | 通信方向 |
| `--access` | Allow / Deny |
| `--protocol` | TCP / UDP |

---

# 🔥 **ASG（Application Security Group）**

---

# 🔰 概要

VMを論理グループ化

---

## 💡 例

```
WebServers
DBServers
```

---

## 🎯 試験ポイント

👉 IPではなくグループ管理可能

---

# ⚠️ **試験の引っかけ**

| 問題 | 正解 |
| --- | --- |
| 数字が大きい方が優先 | ❌ |
| NSGは認証する | ❌ |
| Inboundは送信通信 | ❌ |

---

# 🎯 **まとめ**

- NSG = Azureの通信制御
- 制御内容
    - IP
    - ポート
    - プロトコル
- 超重要
    - **優先順位**
    - **Inbound / Outbound**
    - **サブネット / NIC適用**

---

# 📊 **比較まとめ（Notion対応）**

## ■ Inbound vs Outbound

| 種類 | 内容 |
| --- | --- |
| **Inbound** | 外→内 |
| **Outbound** | 内→外 |

---

## ■ NSG適用場所

| 場所 | 特徴 |
| --- | --- |
| **サブネット** | 全体制御 |
| **NIC** | VM単位 |

---

## ■ よく使うポート

| ポート | 用途 |
| --- | --- |
| `22` | SSH |
| `3389` | RDP |
| `80` | HTTP |
| `443` | HTTPS |

---

## ■ Allow / Deny

| 動作 | 意味 |
| --- | --- |
| **Allow** | 許可 |
| **Deny** | 拒否 |
