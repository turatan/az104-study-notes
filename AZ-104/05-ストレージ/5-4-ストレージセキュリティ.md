# 🔐 **ストレージセキュリティ（Storage Security）**

ここはAZ-104でかなり重要です 👍

特に、

👉 **SAS**

👉 **アクセスキー**

👉 **ネットワーク制御**

👉 **暗号化**

は頻出です。

---

# 🔰 **なぜストレージセキュリティが重要か**

Azure Storageには👇が保存されます

- バックアップ
- 顧客データ
- VMディスク
- 機密ファイル

もし設定ミスすると…

❌ 外部公開

❌ 情報漏洩

❌ ランサムウェア被害

につながります。

---

# 🎯 **試験で重要なセキュリティ機能**

| 機能 | 役割 |
| --- | --- |
| **アクセスキー** | ストレージ認証 |
| **SAS** | 一時アクセス |
| **Entra ID認証** | ユーザー認証 |
| **Firewall** | ネットワーク制御 |
| **Private Endpoint** | 非公開接続 |
| **暗号化** | データ保護 |

---

# 🔑 **アクセスキー（重要）**

---

# 🔰 概要

ストレージアカウント全体の認証キー

---

## 💡 特徴

- 強力
- フルアクセス可能

---

## 🎯 試験ポイント

👉 漏洩時リスク大

---

# 🛠️ **キー表示**

```
az storage account keys list \
--resource-group myRG \
--account-name mystorage123
```

---

# ⚠️ 注意

👉 できるだけ直接利用しない

---

# ⏳ **SAS（超重要）**

---

# 🔰 概要

**Shared Access Signature**

👉 一時的アクセス権

---

## 💡 特徴

- 有効期限設定
- 権限制限可能

---

## 🎯 試験ポイント

👉 外部共有で最重要

---

# 🛠️ **SAS生成**

```
az storage blob generate-sas \
--account-name mystorage123 \
--container-name images \
--name sample.jpg \
--permissions r \
--expiry2026-12-31T23:59:00Z
```

---

# 💡 オプション解説

| オプション | 内容 |
| --- | --- |
| `--permissions r` | 読み取り |
| `--expiry` | 有効期限 |

---

# 🎯 試験ポイント

👉 SAS = 最小権限

---

# 👤 **Entra ID認証**

---

# 🔰 概要

Microsoft Entra IDを使う認証

---

## 💡 メリット

- RBAC利用可能
- キー不要

---

## 🎯 試験ポイント

👉 推奨方式

---

# 🌐 **ストレージファイアウォール**

---

# 🔰 概要

アクセス元IP制限

---

## 💡 できること

- 特定IPのみ許可
- VNet制限

---

## 🎯 試験ポイント

👉 ネットワーク制御で頻出

---

# 🔒 **Private Endpoint（超重要）**

---

# 🔰 概要

Private IPで接続

---

## 💡 イメージ

```
VNet
  ↓
Private Endpoint
  ↓
Storage Account
```

---

## 🎯 試験ポイント

👉 インターネット経由不要

---

# 🔥 **Service Endpointとの違い（超頻出）**

| 項目 | Service Endpoint | Private Endpoint |
| --- | --- | --- |
| IP | Public | Private |
| 接続方式 | Azure Backbone | Private IP |
| セキュリティ | 中 | 高 |

---

## 🎯 超重要

👉 現在は **Private Endpoint推奨**

---

# 🔐 **暗号化（重要）**

---

# ■ 保存時暗号化（Encryption at Rest）

👉 Azure Storageはデフォルト暗号化

---

# ■ 転送時暗号化

👉 HTTPS利用

---

# 🎯 試験ポイント

👉 Azure Storageは既定で暗号化

---

# 🛠️ **HTTPS強制**

```
az storage account update \
--name mystorage123 \
--https-onlytrue
```

---

# 💡 ポイント

- HTTP拒否
- HTTPSのみ許可

---

# 🔄 **キーのローテーション**

## 🔰 概要

定期的なキー変更

---

## 🎯 試験ポイント

👉 セキュリティ強化で重要

---

# ⚠️ **試験の引っかけ**

| 問題 | 正解 |
| --- | --- |
| SASは永続権限 | ❌ |
| Private EndpointはPublic IP使用 | ❌ |
| Azure Storageは暗号化されない | ❌ |

---

# 🎯 **まとめ**

- ストレージセキュリティの中心
    - **アクセスキー**
    - **SAS**
    - **Entra ID**
    - **Private Endpoint**
- 現在の推奨
    - **Entra ID認証**
