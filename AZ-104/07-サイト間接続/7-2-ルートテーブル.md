# 🛣️ **ルートテーブル（Route Table）**

ここはAZ-104ネットワーク分野で重要です 👍

特に、

👉 **通信経路**

👉 **UDR（User Defined Route）**

👉 **Next Hop**

👉 **Firewall連携**

が頻出です。

---

# 🔰 **ルートテーブルとは**

## ■ 概要

**Route Table** とは

👉 **通信の行き先を決める仕組み**

---

## 💡 一言でいうと

👉 **「パケットのナビゲーション」**

---

# 🧠 **イメージ（超重要）**

```
通信
 ↓
Route Table
 ↓
どこへ送るか決定
```

---

## 🎯 試験ポイント

👉 Azureはルートで通信経路を判断

---

# 🌐 **Azureのデフォルトルート**

Azureでは最初からルートがあります。

---

## 💡 例

| 宛先 | 行き先 |
| --- | --- |
| VNet内部 | ローカル |
| インターネット | Internet |
| 不明宛先 | Internet |

---

## 🎯 試験ポイント

👉 何もしなくても基本通信可能

---

# 🔥 **UDR（User Defined Route）**

---

# 🔰 概要

ユーザー定義ルート

👉 管理者が独自ルート作成

---

# 💡 用途

- Firewall経由
- 特定NW経由
- 強制ルーティング

---

# 🎯 試験ポイント

👉 UDR = カスタムルート

---

# 🧠 **ルートの構成要素（超重要）**

| 項目 | 内容 |
| --- | --- |
| **Address Prefix** | 宛先ネットワーク |
| **Next Hop Type** | 次の転送先 |
| **Next Hop IP** | 次ホップIP |

---

# 🎯 試験ポイント

👉 Next Hopが超重要

---

# 🔥 **Next Hop Type（超頻出）**

---

# ① 🌐 **Internet**

👉 インターネットへ

---

# ② 🏠 **Virtual Network**

👉 VNet内部

---

# ③ 🔥 **Virtual Appliance**

👉 Firewallなど

---

# ④ 🚪 **Virtual Network Gateway**

👉 VPN Gateway

---

# 🎯 超重要比較

| Next Hop | 用途 |
| --- | --- |
| **Internet** | 外部 |
| **Virtual Network** | 内部 |
| **Virtual Appliance** | Firewall |
| **VPN Gateway** | VPN |

---

# 🔥 **強制トンネリング（重要）**

---

# 🔰 概要

通信を必ずFirewallへ送る

---

# 💡 イメージ

```
VM
 ↓
Firewall
 ↓
Internet
```

---

## 🎯 試験ポイント

👉 UDR + Virtual Appliance

---

# 📦 **ルートテーブル適用先**

---

# 🔰 概要

サブネットへ関連付け

---

## 💡 イメージ

```
Subnet
  ↓
Route Table
```

---

## 🎯 試験ポイント

👉 VMではなく通常サブネットへ適用

---

# 🛠️ **CLIでルートテーブル作成**

---

## ■ ルートテーブル作成

```
az network route-table create \
--resource-group myRG \
--name myRouteTable
```

---

# 🛠️ **ルート追加**

---

## ■ Firewall経由

```
az network route-table route create \
--resource-group myRG \
--route-table-name myRouteTable \
--name routeToFirewall \
--address-prefix0.0.0.0/0 \
--next-hop-type VirtualAppliance \
--next-hop-ip-address10.0.1.4
```

---

# 💡 オプション解説

| オプション | 内容 |
| --- | --- |
| `--address-prefix` | 宛先 |
| `--next-hop-type` | 次ホップ |
| `--next-hop-ip-address` | 次IP |

---

# 🔗 **サブネット関連付け**

```
az network vnet subnet update \
--resource-group myRG \
--vnet-name myVNet \
--name subnet1 \
--route-table myRouteTable
```

---

# ⚠️ **試験の引っかけ**

| 問題 | 正解 |
| --- | --- |
| UDRはNSG設定 | ❌ |
| Route TableはVMへ直接適用 | ❌ |
| Next Hopは1種類のみ | ❌ |

---

# 🎯 **まとめ**

- Route Table = 通信経路制御
- Azureにはデフォルトルートあり
- UDRで独自制御可能
- 超重要
    - **Next Hop**
    - **Virtual Appliance**
    - **強制トンネリング**

---

| --- | --- |
